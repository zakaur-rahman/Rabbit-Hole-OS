# Synthesis Agents Documentation

Documentation for the AI agent pipeline in `apps/backend/app/services/agents.py` and `apps/backend/app/workers/synthesis_worker.py`.

---

## Pipeline Overview

When a user triggers a synthesis job, it runs as a sequential `arq` background task through 7 agents:

```
Input (query + context + graph edges)
  │
  ▼
1. PlannerAgent      → document outline + section plan
  │
  ▼
2. WriterAgent       → raw Document AST (JSON)
  │
  ▼
3. ReviewerAgent     → validated + hallucination-cleaned AST
  │
  ▼
4. ChartFigureAgent  → figure/chart recommendations injected into AST
  │
  ▼
5. BibNormalizerAgent → deduplicated, cleaned reference list
  │
  ▼
6. CompilationTask   → LaTeX → PDF (non-fatal, optional)
  │
  ▼
7. MemoryAgent       → conceptual pattern extraction for future jobs
  │
  ▼
Output: DocumentAST saved to DB, COMPLETED event published via Redis
```

All agents extend `BaseAgent` and use the same LLM call stack:
- Provider: Gemini 2.5 Flash Lite (primary), DeepSeek V3 / GPT-4o (fallback)
- Retry: 3 attempts with exponential backoff (1s, 2s, 4s)
- All responses must be **valid JSON** (`response_mime_type: application/json`)

## Document AST Schema

The Document AST is the core data structure passed between agents. Defined in `document_ast.py`.

```json
{
  "title": "string",
  "subtitle": "string (optional)",
  "authors": ["string"],
  "date": "string",
  "abstract": "string",
  "sections": [
    {
      "id": "sec-1",
      "title": "string",
      "level": 1,
      "content": [
        {
          "type": "paragraph",
          "data": { "text": "string", "citations": ["1", "2"] }
        },
        {
          "type": "list",
          "data": { "ordered": false, "items": ["item1", "item2"] }
        },
        {
          "type": "table",
          "data": {
            "caption": "string",
            "columns": ["Col A", "Col B"],
            "rows": [["val1", "val2"]],
            "source_refs": ["1"]
          }
        },
        {
          "type": "figure",
          "data": { "path": "", "caption": "string", "source_refs": ["1"] }
        },
        {
          "type": "quote",
          "data": { "text": "string", "source_refs": ["1"] }
        },
        {
          "type": "warning",
          "data": { "text": "string" }
        }
      ],
      "subsections": []
    }
  ],
  "references": [
    {
      "id": "1",
      "title": "string",
      "url": "https://...",
      "authors": ["string"],
      "year": "2024"
    }
  ]
}
```

> **Note:** `id` is auto-generated by `_normalize_ast_data()` if missing. References using `ref_id` are automatically mapped to `id`.

## Agent Reference

### 1. PlannerAgent

**File:** `agents.py:271`  
**Role:** `AI Research Architect`  
**Worker step:** `planner_task` → progress 20%

**Purpose:** Analyzes the research query and context to produce a structured outline. Defines section order, dependencies, and constraints for the Writer.

**Method:** `plan(query, context, relationships)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `query` | Research topic (3–1000 chars) |
| `context` | Source material (truncated to 10,000 chars) |
| `relationships` | Graph edge data (node → node relationships) |

**Example Response:**
```json
{
  "document_outline": [
    {
      "id": "sec-1",
      "title": "Introduction to India",
      "objective": "Provide an overview of India's geography, history, and significance.",
      "ref_ids": ["1", "2"]
    },
    {
      "id": "sec-2",
      "title": "The Himalayas",
      "objective": "Detail the Himalayan mountain range and its role in India's geography.",
      "ref_ids": ["3"]
    },
    {
      "id": "sec-3",
      "title": "Historical Context",
      "objective": "Trace India's independence movement and modern governance.",
      "ref_ids": ["1", "4"]
    }
  ],
  "section_dependencies": {
    "sec-2": ["sec-1"],
    "sec-3": ["sec-1"]
  },
  "reasoning_plan": "Start with broad overview of India, then narrow into geographic features (Himalayas), followed by historical timeline. Use Wikipedia as primary source for geography, Britannica for history.",
  "constraints": [
    "Focus on factual information only",
    "Do not speculate on future political developments",
    "Cite every factual claim"
  ]
}
```

**Failure behavior:** Returns `status: "failed"` with empty `data: {}`. Writer falls back to producing sections without a plan.

### 2. WriterAgent

**File:** `agents.py:328`  
**Role:** `Research Synthesis Writer`  
**Worker step:** `writer_task` → progress 40%

**Purpose:** Produces the full Document AST based on the Planner's outline. Inserts inline citations and structures content into typed blocks.

**Method:** `write(query, context, plan, references_json)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `query` | Research topic |
| `context` | Full source material (truncated to 20,000 chars) |
| `plan` | Output from PlannerAgent |
| `references_json` | Pre-built reference map `{"[1]": {"url": "...", "title": "..."}}` |

**Example Response:**
```json
{
  "title": "India: Geography, History, and Culture",
  "abstract": "This document synthesizes research on India covering its geographical features, historical milestones, and regional diversity. Key sources include Wikipedia and Britannica.",
  "sections": [
    {
      "title": "Introduction to India",
      "content": [
        {
          "type": "paragraph",
          "data": {
            "text": "India is the world's most populous country and the seventh-largest by area, located in South Asia. [1] It borders Pakistan to the northwest, China and Nepal to the north, and Bangladesh and Myanmar to the east. [1]",
            "citations": ["1"]
          }
        },
        {
          "type": "list",
          "data": {
            "ordered": false,
            "items": [
              "Area: 3.29 million km²",
              "Population: 1.44 billion (2024)",
              "Capital: New Delhi",
              "Languages: Hindi, English + 21 official languages"
            ]
          }
        }
      ]
    },
    {
      "title": "The Himalayas",
      "content": [
        {
          "type": "paragraph",
          "data": {
            "text": "The Himalayas form the northern boundary of India and contain the world's highest peaks. [3] Mount Everest, at 8,849 metres, lies on the Nepal-Tibet border. [3]",
            "citations": ["3"]
          }
        },
        {
          "type": "figure",
          "data": {
            "path": "",
            "caption": "Himalayan mountain range viewed from northern India.",
            "source_refs": ["3"]
          }
        }
      ]
    }
  ],
  "references": [
    { "ref_id": "1", "url": "https://en.wikipedia.org/wiki/India", "title": "India" },
    { "ref_id": "3", "url": "https://www.britannica.com/place/Himalayas", "title": "Himalayas | Mountains, Asia" }
  ]
}
```

> **Note:** The schema uses `ref_id` — this is automatically normalized to `id` by `_normalize_ast_data()` before Pydantic validation.

**Failure behavior:** Returns `status: "failed"` with empty `data: {}`. Pipeline aborts (fatal step).

### 3. ReviewerAgent

**File:** `agents.py:394`  
**Role:** `Academic Validation Agent`  
**Worker step:** `reviewer_task` → progress 60%

**Purpose:** Validates the Writer's AST output. Detects hallucinations (content not grounded in context), missing citations, duplicate headings, and structural issues. Returns a corrected AST.

**Method:** `review(ast, context)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `ast` | Full Document AST from WriterAgent |
| `context` | Original source material (truncated to 10,000 chars) |

**Example Response:**

The Reviewer returns the same AST schema as the Writer, with corrections applied:

```json
{
  "title": "India: Geography, History, and Culture",
  "abstract": "...",
  "sections": [
    {
      "title": "Introduction to India",
      "content": [
        {
          "type": "paragraph",
          "data": {
            "text": "India is the world's most populous country [1] and the seventh-largest by area, located in South Asia. [1]",
            "citations": ["1"]
          }
        }
      ]
    }
  ],
  "references": [...]
}
```

Typical corrections made:
- Removed claim: *"India has the world's fastest growing tech sector"* — not grounded in provided context
- Added citation `[2]` to paragraph referencing Delhi
- Merged two sections both titled "History"

**Failure behavior:** Returns original AST unchanged (`data: ast`). Pipeline continues with unreviewed AST.

### 4. ChartFigureAgent

**File:** `agents.py:433`  
**Role:** `Visual Reasoning Agent`  
**Worker step:** `chart_task` → progress 75%

**Purpose:** Scans the AST for numeric, comparative, or temporal data and recommends data visualizations (bar charts, line graphs, tables) to enrich the document.

**Method:** `analyze(ast)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `ast` | Full Document AST from ReviewerAgent |

**Example Response:**
```json
{
  "figures": [
    {
      "type": "bar",
      "caption": "Population comparison of India's top 5 cities (2024)",
      "section_id": "sec-2",
      "data": {
        "labels": ["Mumbai", "Delhi", "Bangalore", "Chennai", "Hyderabad"],
        "values": [20.7, 33.8, 13.6, 10.9, 10.5],
        "unit": "million"
      }
    },
    {
      "type": "table",
      "caption": "India's GDP growth by decade",
      "section_id": "sec-4",
      "data": {
        "columns": ["Decade", "GDP (USD Trillion)", "Growth %"],
        "rows": [
          ["1990s", "0.48", "5.7%"],
          ["2000s", "1.06", "7.1%"],
          ["2010s", "2.26", "6.9%"]
        ]
      }
    }
  ]
}
```

**Failure behavior:** Returns `data: {"figures": []}`. Pipeline continues — figures are simply omitted.

### 5. BibNormalizerAgent

**File:** `agents.py:468`  
**Role:** `Citation & Reference Engine`  
**Worker step:** `bib_task` → progress 85%

**Purpose:** Deduplicates references, cleans titles, and assigns consistent `ref_id`s across the document. Ensures every URL used in the AST maps to exactly one canonical reference.

**Method:** `normalize(ast, references)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `ast` | Full Document AST with embedded reference stubs |
| `references` | Raw reference list from source map |

**Example Response:**
```json
{
  "normalized_references": [
    {
      "ref_id": "R1",
      "url": "https://en.wikipedia.org/wiki/India",
      "title": "India"
    },
    {
      "ref_id": "R2",
      "url": "https://www.britannica.com/place/Delhi",
      "title": "Delhi"
    },
    {
      "ref_id": "R3",
      "url": "https://www.britannica.com/place/Himalayas",
      "title": "Himalayas | Mountains, Asia"
    },
    {
      "ref_id": "R4",
      "url": "https://hi.wikipedia.org/wiki/Bihar",
      "title": "Bihar"
    }
  ]
}
```

> **Note:** The LLM sometimes returns a raw array `[{...}, ...]` instead of the wrapper object. This is automatically detected and wrapped: `{"normalized_references": [...]}`.

**Failure behavior:** Returns original `references` list unchanged. Pipeline continues.

### 6. MemoryAgent

**File:** `agents.py:515`  
**Role:** `Cross-Document Learning Engine`  
**Worker step:** `memory_task` → progress N/A (non-fatal)

**Purpose:** Extracts conceptual patterns and user preferences from completed documents. Intended for future personalization — e.g., preferred section ordering, frequently referenced domains.

**Method:** `learn(query, ast)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `query` | Original research topic |
| `ast` | Final completed Document AST |

**Example Response:**
```json
{
  "main_concepts": ["India", "Himalayas", "Bihar", "Delhi", "South Asia"],
  "relationships": [
    { "from": "India", "to": "Himalayas", "relation": "contains" },
    { "from": "India", "to": "Bihar", "relation": "has_state" }
  ],
  "structural_preferences": {
    "preferred_section_count": 6,
    "uses_tables": true,
    "uses_figures": true,
    "citation_density": "high"
  },
  "domain_affinity": ["geography", "history", "political-science"]
}
```

**Failure behavior:** Non-fatal. Pipeline marks COMPLETED regardless. Memory data is discarded silently.

### 7. RecoveryAgent

**File:** `agents.py:552`  
**Role:** `Stability & Recovery Engine`  
**Worker step:** Called from `compilation_task` on LaTeX errors

**Purpose:** Diagnoses LaTeX compilation failures and proposes recovery strategies. Currently used when PDF compilation fails — the agent analyzes the error and provides instructions (though automatic patching is a TODO).

**Method:** `diagnose(error, failed_agent, last_output)`

**Prompt inputs:**
| Input | Description |
|-------|-------------|
| `error` | Error message / LaTeX error log |
| `failed_agent` | Name of the agent/step that failed (e.g., `"LatexCompiler"`) |
| `last_output` | Last 2,000 chars of the failing agent's output |

**Example Response:**
```json
{
  "retry_action": "retry",
  "root_cause": "Unescaped special character '&' in table cell at section 3.",
  "instruction_tweak": "Escape all '&' characters in table rows as '\\&'. Also check for unmatched curly braces in section titles.",
  "confidence": 0.85
}
```

Possible `retry_action` values:
| Value | Meaning |
|-------|---------|
| `"retry"` | Agent believes a fix is possible, provides instructions |
| `"stop"` | Agent cannot diagnose, pipeline should abort compilation |

**Failure behavior:** Returns `data: {"retry_action": "stop"}`. Compilation loop breaks.

## BaseAgent Utilities

All agents share these methods from `BaseAgent`:

### `_call_llm_with_retry(prompt, system_instruction, temperature=0.1)`
Calls the configured LLM provider with automatic retry on transient errors:
- `429` (rate limit), `500`, `502`, `503`, `504` → retry with backoff
- `ConnectError`, `TimeoutException` → retry with backoff
- Max 3 attempts (1s → 2s → 4s delays)

### `_parse_llm_json(raw_response, expected_keys=None)`
Parses LLM output as JSON with 3 levels of recovery:
1. Direct `json.loads()`
2. Replace `'` with `"` and retry
3. `raw_decode()` to strip trailing text after valid JSON

### `_clean_json_response(raw_text)`
Strips markdown code fences (` ```json `) and extracts the first JSON object `{}` or array `[]`.

### `_smart_truncate(text, max_chars)`
Truncates text at sentence boundaries (`.`) rather than mid-word. Appends `[... content truncated ...]`.

### `_validate_input(value, name, min_length, max_length, allow_empty)`
Validates and HTML-unescapes string inputs before passing to LLM.

## Error Handling Summary

| Agent | Fatal? | Failure Output |
|-------|--------|---------------|
| PlannerAgent | No (Writer proceeds without plan) | `{}` |
| WriterAgent | **Yes** — pipeline aborts | `{}` |
| ReviewerAgent | No — original AST passed through | original `ast` |
| ChartFigureAgent | No | `{"figures": []}` |
| BibNormalizerAgent | No — raw refs passed through | original `references` |
| CompilationTask | No (non-fatal since v0.2) | LaTeX step skipped |
| MemoryAgent | No (non-fatal since v0.2) | discarded |
| RecoveryAgent | No | `{"retry_action": "stop"}` |

## LLM Provider Configuration

Configured in `llm.py` via `get_ai_client()`. Priority order:

1. **Gemini** (`GEMINI_API_KEY`) — model: `gemini-2.5-flash-lite`, `response_mime_type: application/json`
2. **Hugging Face** (`HF_TOKEN`)
3. **Chutes.ai** (`CHUTES_API_KEY`) — model: `deepseek-ai/DeepSeek-V3-0324`
4. **OpenAI** (`OPENAI_API_KEY`) — model: `gpt-4o`
5. **Mock** (no keys found) — returns `"{}"` for all calls

Keys are loaded via Pydantic `Settings` from `.env` (not `os.environ` directly).
